# Claude Code 安装、配置与运行手册

本手册用于快速完成 Claude Code 的安装、初始化配置和日常使用。

## 1. 功能简介

Claude Code 是面向开发场景的 AI 助手工具，常用于：

- 阅读与理解代码库
- 生成功能代码与脚本
- 修复缺陷并补充测试
- 生成技术文档与变更说明

## 2. 环境准备

开始前请确认：

- 已安装 Git
- 已安装 Node.js（建议 18+）
- 网络可访问相关服务
- 已准备可用账号或 API 凭据（按你的团队接入方式）

检查命令：

```bash
git --version
node -v
```

## 3. 安装 Claude Code

> 不同团队的安装源可能不同，以下为通用流程。

## 3.1 使用 npm 全局安装（常见）

```bash
npm install -g <claude-code-package-name>
```

安装后验证：

```bash
claude --version
```

## 3.2 桌面端安装（如团队使用桌面版）

1. 下载并安装 Claude Code Desktop。
2. 启动后选择本地工作目录。
3. 在设置页绑定账号或 API 凭据。

## 4. 配置

## 4.1 配置凭据（环境变量示例）

### Windows PowerShell（当前会话）

```powershell
$env:ANTHROPIC_API_KEY="你的_api_key"
```

### macOS / Linux（当前会话）

```bash
export ANTHROPIC_API_KEY="你的_api_key"
```

如需长期生效，请写入 shell 配置文件（如 `.zshrc`、`.bashrc`）。

## 4.2 常见配置项

根据团队规范，通常还会设置：

- 默认模型
- 输出语言（中文/英文）
- 默认工作目录
- 代码修改权限策略（只读 / 可编辑）

## 5. 运行与基本用法

## 5.1 在项目目录启动

```bash
cd your-project
claude
```

## 5.2 推荐交互流程

1. 先让它熟悉项目：

```text
熟悉一下这个项目结构和启动方式
```

2. 给出明确任务：

```text
帮我定位登录流程里的空指针问题并修复
```

3. 要求验证：

```text
运行测试并告诉我失败原因和修复建议
```

4. 输出总结：

```text
总结改动文件、风险点和回滚方案
```

## 6. 最佳实践

- 任务描述尽量具体（目标、范围、约束）
- 先读后改，避免直接大改
- 小步提交，便于回滚
- 每次改动后做构建和关键路径验证

## 7. 常见问题排查

## 7.1 命令找不到

- 检查全局安装是否成功
- 检查 PATH 是否包含 npm 全局目录

## 7.2 凭据校验失败

- 检查环境变量名和值
- 检查是否在当前终端会话生效

## 7.3 网络连接失败

- 检查代理、防火墙和公司网络策略
- 必要时切换网络后重试

## 7.4 构建或测试失败

- 先看首个报错点
- 核对 Node.js 与依赖版本
- 清理依赖后重装再试

## 8. 快速检查清单

- [ ] `claude --version` 可用
- [ ] 凭据已配置并生效
- [ ] 能在项目目录正常启动
- [ ] 能完成一次“读代码 -> 改代码 -> 验证”闭环

完成以上项后，可把 Claude Code 纳入日常开发流程。
